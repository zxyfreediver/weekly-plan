# 一键导入上周未完成 — 设计文档

## 一、需求概述

1. **一键导入**：可将上周未完成的主任务、子任务、进度导入到本周
2. **数据分离**：导入后生成独立副本，本周的修改不影响上周原始数据
3. **AI 总结去重**：选择多周做总结时，同一逻辑任务只算一次

---

## 二、已确认设计决策

| 项 | 决策 |
|----|------|
| 子任务/进度复制范围 | **只复制未完成** |
| AI 总结代表选择 | **B：取 week_start 最晚的**（优先本周） |
| 按钮显示 | **始终显示**，有提示文案 |
| 本周已有任务时 | **与导入任务合并**，不覆盖 |
| 导入后展开 | **不自动展开**，用户手动展开 |

---

## 三、数据模型扩展

### 新增溯源字段

| 表 | 字段 | 类型 | 说明 |
|----|------|------|------|
| tasks | source_task_id | TEXT (可空) | 导入时指向源任务 id |
| sub_tasks | source_sub_task_id | TEXT (可空) | 导入时指向源子任务 id |
| sub_task_progress | source_progress_id | TEXT (可空) | 导入时指向源进度 id |

- 用户手动创建：`source_*` 为空
- 从上周导入：新记录写入对应 `source_*`，指向上周原始记录

---

## 四、导入逻辑

### 触发条件

- 按钮**始终显示**
- 点击时：若上周无未完成主任务，提示「上周暂无未完成任务」；否则执行导入

### 导入范围

- **主任务**：只导入 `is_completed = false` 的主任务
- **子任务**：只复制**未完成**的子任务（`is_completed = false`）
- **进度**：只复制**未完成**的进度（`is_completed = false`）

### 合并规则（本周已有任务时）

- 不覆盖本周已有任务
- 只导入「上周有、本周没有」的主任务（按 `source_task_id` 或内容匹配判断是否已存在）
- 判断逻辑：上周主任务 A，若本周已存在 `source_task_id = A.id` 的任务，则跳过 A；否则导入 A 及其未完成子任务、进度

### 导入流程（伪代码）

```
1. 查出上周 (week_start = 上周一) 所有 is_completed=false 的主任务
2. 对每个主任务 t：
   a. 检查本周是否已有 source_task_id = t.id 的任务 → 有则跳过
   b. 创建新主任务 t'：复制 content/description，week_start=本周一，source_task_id=t.id
   c. 对 t 的每个未完成子任务 st：
      - 创建新子任务 st'：复制，task_id=t'.id，source_sub_task_id=st.id
      - 对 st 的每个未完成进度 p：
         - 创建新进度 p'：复制，sub_task_id=st'.id，source_progress_id=p.id
3. 返回导入数量
```

## 五、AI 总结去重

### 分组规则

- 按**溯源根**分组：沿 `source_task_id` 向上追溯，直到无 `source_task_id` 的主任务
- 同一溯源根下的所有副本 = 同一逻辑任务

### 代表选择（策略 B）

- 每组内，在时间范围内、有完成项的副本中，取 **week_start 最晚** 的一条作为代表
- 即优先用本周的完成状态

## 六、UI 行为

### 按钮

- **位置**：周选择器区域（与「一键展开」「不展示已完成」同排）
- **文案**：「导入上周未完成」
- **提示**（如 tooltip 或旁白）：
  - 默认：「将上周未完成的主任务、子任务、进度复制到本周，与本周任务合并」
  - 上周无未完成时：点击后 Toast「上周暂无未完成任务」

### 导入后

- 不自动展开，用户手动展开查看

## 八、边界情况

1. **上周无主任务**：提示「上周暂无未完成任务」，不执行导入
2. **上周主任务已完成，但有未完成子任务**：主任务不导入（主任务已完成即整体视为完成）
3. **本周已有从上周导入的同一任务**：通过 `source_task_id` 去重，不重复导入
4. **多周连续导入**：溯源链 a1←a2←a3，溯源根仍为 a1，去重逻辑不变
